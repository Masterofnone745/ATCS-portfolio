4x4 XOR Toggle Light Puzzle

This project implements a classic "Lights Out" style puzzle on an Arduino, where the state of four LEDs is controlled by four buttons using **XOR (toggle) logic**. The challenge was to ensure mathematical solvability through bitwise manipulation.

Technical Deep Dive: Bitwise XOR

The core of the puzzle relies on the Bitwise XOR operator (^). This operator allows us to efficiently toggle the state of specific LEDs (bits) in a single operation.

The XOR Mechanism

  • • Puzzle state held in one 4-bit integer, boardState.
  • • Each button has a unique **Pattern Mask**.
  • • Toggle calculation: boardState = boardState ^ mask;

Ensuring Solvability

Choosing the right masks so that the resulting matrix of button operations is **invertible**. The following set of masks guarantees that every state can return to zero.

Button Decimal Binary Mask Toggles
B1 14 1110 LEDs 4, 3, & 2
B2 12 1100 LEDs 4 & 3
B3 3 0011 LEDs 2 & 1
B4 7 0111 LEDs 3, 2, & 1

Key Specs

  • Debouncing: 50ms software delay to filter mechanical noise.
  • Randomization: Analog noise (A0) seeding for true randomness.
  • Physical Output: Bit-shifting (>>) used to map integer to 4 GPIO pins.

void updateLeds() {
  digitalWrite(ledPin1, (boardState >> 0) & 0x01); // LSB
  digitalWrite(ledPin2, (boardState >> 1) & 0x01); 
  digitalWrite(ledPin3, (boardState >> 2) & 0x01); 
  digitalWrite(ledPin4, (boardState >> 3) & 0x01); // MSB
}
